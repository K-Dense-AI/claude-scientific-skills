# agent-guardian

에이전트 적합성 점검 스킬. 스킬 작성/수정 시 자동 활성화.
점검 에이전트(체커)와 구현 에이전트(모디파이어) 2개로 운영.

## 판단 기준

### Agent 필요 (아래 중 하나라도 해당)
- 여러 소스 병렬 조회 (PubMed, GitHub, DB 등)
- 결과물이 크고 오래 걸리는 작업 (논문 섹션, 종합 리뷰)
- 컨텍스트 오염 우려 (대량 중간 결과)
- 자율 탐색/판단이 필요한 작업
- 다수 파일 수정이 포함된 작업

### Agent 불필요 (직접 실행)
- 사용자와 중간 확인이 필요한 인터랙티브 작업
- 단순 참조/조회 (단일 API 호출)
- 짧고 즉각적인 작업
- 설계 과정에 유저 판단이 필요한 작업

---

## 실행 방식

**반드시 2개의 Agent를 순차 실행:**

### 1단계 — [체커 에이전트]

```
목적: 모든 SKILL.md 읽고 에이전트 필요 여부 판단

작업:
1. C:\Users\Jahyun\claude-scientific-skills\scientific-skills\ 하위 모든 SKILL.md 목록 수집
2. 각 SKILL.md 읽기 (병렬)
3. 판단 기준에 따라 분류:
   - needs_agent: [] (Agent 실행 필요 스킬)
   - no_agent: []   (직접 실행 스킬)
   - already_set: [] (이미 실행 방식 명시된 스킬)
4. 분류 결과 반환
```

### 2단계 — [모디파이어 에이전트]

```
목적: needs_agent 목록의 SKILL.md 상단에 실행 방식 섹션 추가

추가할 내용 (SKILL.md 최상단 또는 첫 번째 ## 섹션 직전):
---
## 실행 방식
이 스킬의 모든 작업은 반드시 Agent tool을 사용해 서브에이전트로 실행할 것.
직접 실행하지 말고, 작업 전체를 Agent에 위임한다.
---

no_agent 목록은 수정하지 않음.
already_set 목록도 수정하지 않음.

작업 후 수정된 파일 목록 반환.
```
