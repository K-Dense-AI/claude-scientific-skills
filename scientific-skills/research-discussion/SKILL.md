# Research Discussion (연구 디스커션)

실험 결과 해석, 연구 논의, 디스커션 기록을 위한 스킬.

## 핵심 기능

1. **결과 해석** — 실험 데이터의 의미 분석 및 해석
2. **비교 분석** — 문헌 데이터와의 비교, 차이점 논의
3. **논리 구조화** — Discussion 섹션 작성을 위한 논리 흐름 설계
4. **디스커션 기록** — 논의 내용, 결론, 후속 과제 기록
5. **체인 모드** — 해석→팩트체크→비교→구조화를 자동 연결하는 파이프라인

## 워크플로우

```
사용자 요청 분석
    ↓
모드 판별: [해석] / [비교] / [구조화] / [기록] / [체인]
    ↓
┌─ 단일 모드: 해당 모드 실행
└─ 체인 모드: 해석 → 팩트체크(research-assistant) → 비교 → 구조화 자동 연결
    ↓
사실관계 확인 필요 시 → research-assistant 팩트체크 자동 호출
    ↓
실행 & persistent storage 저장
    ↓
실험 이력(experiment-manager)과 연동 가능
```

### 모드 자동 전환 규칙

디스커션 진행 중 다음 조건 충족 시 모드가 자동 전환된다:

| 현재 모드 | 전환 조건 | 전환 대상 |
|-----------|-----------|-----------|
| 해석 | 해석 중 수치/주장의 정확성 의문 | → 팩트체크 (research-assistant) |
| 해석 | 해석 완료 후 "문헌에서는?" 질문 | → 비교 분석 |
| 비교 | 비교 중 문헌 수치 불일치 발견 | → 팩트체크 (research-assistant) |
| 비교 | 비교 완료 후 "논문에 쓰려면?" | → 논리 구조화 |
| 구조화 | Discussion 작성 중 근거 부족 | → 팩트체크 (research-assistant) |
| 모든 모드 | 세션 종료 시 | → 기록 (자동 저장) |

## 모드 1: 결과 해석 (Interpret)

### 트리거
- "이 결과 어떻게 해석해야 할까?" (→ 메커니즘/맥락 중심 해석)
- "이 데이터가 의미하는 게 뭐야?"
- "왜 이런 결과가 나왔을까?" (→ Interaction Matrix 기반 분석)

> **분기 기준** (COMMONS.md 참조): 메커니즘/맥락/문헌 중심 해석은 여기(M1), 데이터/추세선/통계 중심 분석은 SKILL-1 M6.

### 실행 절차

1. **데이터 확인**: 사용자가 제공한 실험 결과 파악
2. **맥락 수집** (필수): 아래 체크리스트 확인 → 미확인 항목은 사용자에게 질문
3. **해석 제시**: 가능한 해석들을 근거와 함께 제시
4. **추가 검증 제안**: 해석을 확인하기 위한 추가 실험 제안

### 맥락 수집 필수 체크리스트

해석 전에 **반드시** 아래 항목을 확인한다. 사용자가 제공하지 않은 항목은 가정하지 말고 질문한다.

```
[필수 확인 항목]
├─ 반응 온도: ___°C
├─ pH / 버퍼 종류·농도: ___
├─ 반응 시간: ___ h
├─ 기질 종류·농도: ___
├─ 효소 종류·농도 (각 효소별): ___
├─ 보조인자/공기질 (ATP, NAD⁺, 금속이온, AcP 등): ___
├─ 반응 부피/스케일: ___
└─ 기존 이력: storage 조회 (exp:log, disc:session)

[선택 확인 항목 — 해당 시]
├─ 산소 조건 (호기/혐기): ___
├─ 효소 유래 (균주/재조합): ___
├─ 정제 수준 (crude/purified): ___
└─ 분석 방법 (HPLC, UV, etc.): ___
```

**원칙**: 온도, pH, 기질·효소 농도를 모르면 해석의 정확도가 크게 떨어진다. 가정 금지.

### 생명공학 실험 해석 체크리스트

**기본 항목:**
- **효소 반응**: 전환율 변화 원인 (기질 억제, 생성물 억제, pH 변화, 효소 실활)
- **전세포 촉매**: 세포 생존율, 투과성, 대사 경쟁, 독성 영향
- **HPLC 분석**: 피크 변화, 체류시간 이동, 신규 피크, 정량적 변화
- **단백질 발현**: 발현량, 가용성, 활성, 폴딩

### 반응계 물질 상호작용 분석 (Interaction Matrix)

해석 시 반응계에 존재하는 **모든 물질 간 상호작용**을 체계적으로 스캔한다. 단일 원인만 찾지 말고, 물질 간 복합 작용을 반드시 고려한다.

**분석 절차:**
```
Step 1: 반응계 물질 목록 작성
  ├─ 기질 (substrate)
  ├─ 생성물 (product)
  ├─ 효소 (enzyme) — 각각
  ├─ 보조인자/공기질 (ATP, NAD⁺, AcP, PEP 등)
  ├─ 금속이온 (Mg²⁺, Mn²⁺, Co²⁺ 등)
  ├─ 버퍼 성분
  └─ 기타 첨가제

Step 2: 상호작용 매트릭스 작성 (아래 5개 카테고리)

Step 3: 시간 의존성 평가 — 반응 진행에 따라
        물질 조성이 변하면 상호작용도 변함
```

**카테고리 1: 화학적 안정성 / 분해**
```
각 물질이 반응 조건(온도, pH, 금속이온)에서 안정한가?
├─ 자발적 가수분해: AcP, sugar phosphate, ATP 등
│  · AcP: 중성 pH 반감기 수 시간, Mg²⁺·고온 촉진
│  · Sugar phosphate: 산성 pH에서 가수분해 가속
│  · ATP: Mg²⁺ 존재 시 비교적 안정, 고온에서 분해
├─ 이성화/라세미화: 당류의 epimerization, 아미노산 라세미화
├─ 산화: 금속이온 촉매 산화, 공기 접촉 산화 (DTT, 티올 등)
└─ 금속이온에 의한 촉진/억제: Mg²⁺가 AcP 분해 촉진,
   Ca²⁺가 효소 구조 안정화 등
```

**카테고리 2: 물질-효소 비효소적 변형**
```
반응계 물질이 효소를 화학적으로 변형하는가?
├─ 아세틸화: AcP, AcCoA → Lys ε-amino기 비효소적 아세틸화
│  · 활성부위 Lys 변형 시 비가역적 실활
│  · in vitro에서는 deacetylase 부재 → 누적
│  · 인산기 결합부위에 AcP 직접 결합 → 기질경쟁 + 아세틸화
├─ 인산화: AcP → His, Asp 잔기 인산화
├─ 글리케이션: sugar, sugar phosphate → Lys과 Schiff base
├─ 산화적 변형: H₂O₂, ROS → Cys, Met 산화
└─ 킬레이션: EDTA, 시트르산이 필수 금속이온 제거 → 실활
```

**카테고리 3: 물질-물질 직접 반응 / 용해도**
```
물질끼리 비효소적으로 반응하거나 침전하는가?
├─ 금속-인산 침전: Mg²⁺/Ca²⁺ + Pi → 불용성 인산염
│  · 반응 중 Pi 축적 시 금속이온 격리 → 효소 활성 저하
├─ 금속-기질 킬레이트: 당 인산 + 금속이온 → 킬레이트
│  · 유효 기질 농도 감소
├─ 기질 간 비효소적 반응: 당 + 아민 → 갈변(Maillard)
│  · 장시간 고온 반응에서 고려
├─ pH 변화 유발: acetate 생성 → pH 저하
│  · 약한 버퍼에서 유의미
└─ 용해도 한계: 고농도 기질/생성물 → 석출
   · sugar phosphate, 보조인자 결정화
```

**카테고리 4: 억제 (Inhibition)**
```
물질이 효소를 직접 억제하는가?
├─ 기질 억제: 고농도 기질이 역으로 활성 저하
├─ 생성물 억제: 생성물이 활성부위에 결합
├─ 보조인자 부산물 억제: ADP가 ATP 결합부위 경쟁,
│  NAD⁺/NADH 비율 변화
├─ 비경쟁/반경쟁 억제: 알로스테릭 부위 결합
│  · ATP, PEP, OAA 등이 알로스테릭 효과를 가질 수 있음
├─ 역반응 평형: 가역 반응에서 생성물 축적 → 평형 역전
│  · XylA (xylose ⇌ xylulose), AckA (AcP+ADP ⇌ ATP+acetate)
└─ Dead-end 억제: 기질 유사체가 활성부위 점유
```

**카테고리 5: 시간 의존적 변화**
```
반응 진행에 따라 상호작용 양상이 어떻게 바뀌는가?
├─ 초기 (0-30min): 기질 충분, 보조인자 안정 → 정상 반응
├─ 중기 (30min-2h): AcP 분해 시작, 아세틸화 누적,
│  acetate 축적, pH 변화 시작
├─ 후기 (2h+): 보조인자 고갈, 효소 변형 포화,
│  평형 도달, 침전물 형성 가능
└─ 시점별 핵심 질문:
   · "어떤 물질이 먼저 고갈/분해되는가?"
   · "어떤 부산물이 가장 빠르게 축적되는가?"
   · "시간에 따라 어떤 새로운 상호작용이 발생하는가?"
```

### 적용 규칙

1. **항상 적용**: 반응계에 3종 이상 물질이 존재하는 모든 in vitro 실험
2. **간이 적용**: 단일 효소 반응 — 카테고리 2, 4만 체크
3. **전체 적용**: 다효소 캐스케이드, ATP 재생 시스템 포함 반응
4. **시간축 필수**: 반응 정지/속도 변화 문제 시 카테고리 5 반드시 수행
5. **검색 키워드**: 상호작용이 의심되면 `{물질A} {물질B} inhibition OR inactivation OR stability OR interaction`으로 문헌 검색

## 모드 2: 비교 분석 (Compare)

### 트리거
- "문헌이랑 비교해줘" (→ 내 결과 vs 문헌 데이터)
- "다른 연구 결과랑 어떻게 달라?"
- "이 수율이 높은 편이야?"

> **분기 기준** (COMMONS.md 참조): 문헌 대비 비교는 여기(M2), 내 실험끼리 비교는 SKILL-1 M7.

### 실행 절차

1. **비교 대상 설정**: 사용자 결과 + 문헌 데이터
2. **문헌 검색**: 유사 시스템의 보고된 결과 탐색 (web_search)
   - 기본 검색: {효소명} + {반응/기질} + {조건 키워드}
   - **상호작용 검색** (Interaction Matrix에서 위험 식별 시):
     `{물질A} {물질B} inhibition OR inactivation OR stability OR interaction`
     예) "acetyl phosphate xylulokinase inactivation"
     예) "magnesium phosphate precipitation enzyme cascade"
3. **비교표 생성**: 조건별, 결과별 체계적 비교
4. **차이 분석**: 차이의 원인 논의 (화학적 상호작용 포함)

### 비교 분석 형식

```
═══════════════════════════════════════════════════
           비교 분석

──────────────────────────────────────────────────

├─ 우위 요소: [what's better and why]

═══════════════════════════════════════════════════
```

## 모드 3: 논리 구조화 (Structure)

### 트리거
- "Discussion 어떻게 쓸까?"
- "논의 구조 잡아줘"
- "결과 정리해서 논문에 넣을 수 있게 해줘"

### Discussion 논리 구조

```
═══════════════════════════════════════════════════
           Discussion 논리 구조
═══════════════════════════════════════════════════

1. 핵심 발견 요약 (Opening Statement)
   "본 연구에서 [main finding]을 확인하였다."

2. 결과 해석 (Interpretation)
   ├─ 주요 결과 1의 의미
   │   ├─ 데이터 근거
   │   └─ 메커니즘적 설명
   ├─ 주요 결과 2의 의미
   └─ 결과 간 연결성

3. 문헌 비교 (Comparison with Literature)
   ├─ 일치하는 보고: (Author, Year) — DOI 링크
   ├─ 상이한 보고: (Author, Year) → 차이 원인 — DOI 링크
   └─ 본 연구의 차별점

4. 기여 & 의의 (Significance)
   ├─ 학술적 기여: [contribution]
   ├─ 실용적 의의: [practical impact]
   └─ 기존 지식 확장: [how this extends knowledge]

5. 한계점 (Limitations)
   ├─ 실험적 한계: [limitations]
   └─ 해석의 한계: [caveats]

6. 향후 연구 (Future Directions)
   ├─ 단기 과제: [immediate next steps]
   └─ 장기 과제: [long-term goals]

7. 결론 (Concluding Statement)
   [1-2문장 핵심 메시지]

──────────────────────────────────────────────────
논리 흐름 점검:
├─ 각 단락이 자연스럽게 연결되는가? [Y/N]
├─ 모든 주요 결과가 논의되었는가? [Y/N]
├─ 과잉 해석은 없는가? [Y/N]
└─ 결론이 데이터에 의해 지지되는가? [Y/N]
```

## 모드 5: 체인 모드 (Chain)

### 트리거
- "결과 해석부터 Discussion까지 한 번에 해줘"
- "논문용으로 전체 분석해줘"
- "이 데이터 종합 분석"
- 명시적으로 "체인" 또는 "파이프라인" 언급 시

### 실행 절차 (자동 파이프라인)

```
[Step 1] 결과 해석 (Interpret)
    │  사용자 데이터 → 해석 제시
    │
    ▼ 해석 중 수치/주장 발견 시 자동 트리거
[Step 2] 사실관계 확인 (Fact-Check → research-assistant 연동)
    │  해석에서 언급된 수치/조건을 문헌으로 교차 검증
    │  research-assistant의 팩트체크 모드 호출
    │  결과: ✅ 확인 / ⚠️ 부분확인 / ❌ 불일치
    │
    ▼ 검증된 데이터로
[Step 3] 비교 분석 (Compare)
    │  팩트체크에서 수집된 문헌 + 추가 검색으로 비교표 생성
    │
    ▼ 비교 결과를 바탕으로
[Step 4] 논리 구조화 (Structure)
    │  Discussion 7단계 구조 생성
    │  논리 흐름 점검 체크리스트 수행
    │
    ▼ 자동
[Step 5] 기록 (Record)
    │  전체 체인 결과를 persistent storage에 저장
    └→ disc:chain:{date}:{id} 키로 저장
```

### 체인 모드 출력 형식

```
═══════════════════════════════════════════════════
           종합 분석 보고서 (Chain Mode)
═══════════════════════════════════════════════════

분석일: [date]
파이프라인: 해석 → 팩트체크 → 비교 → 구조화

──────────────────────────────────────────────────
STEP 1. 결과 해석
──────────────────────────────────────────────────
[해석 결과 — 모드 1 형식 준용]

──────────────────────────────────────────────────
STEP 2. 사실관계 확인
──────────────────────────────────────────────────
확인 항목 1: [claim/value]
  판정: ✅/⚠️/❌
  근거: (Author, Year) — DOI 링크
확인 항목 2: [claim/value]
  판정: ✅/⚠️/❌
  근거: (Author, Year) — DOI 링크

──────────────────────────────────────────────────

──────────────────────────────────────────────────

──────────────────────────────────────────────────

──────────────────────────────────────────────────
[전체 파이프라인에서 도출된 핵심 결론]

═══════════════════════════════════════════════════
```

### 팩트체크 자동 호출 규칙

체인 모드에서 research-assistant의 팩트체크 모드가 자동 호출되는 조건:

1. **수치 검증**: 해석 중 구체적 수치(전환율, Km, kcat, 온도, pH 등)가 언급될 때
2. **조건 검증**: "~조건에서 ~효소가 최적"과 같은 주장이 나올 때
3. **메커니즘 검증**: 제시한 메커니즘적 설명이 문헌에 근거하는지 확인
4. **비교 데이터 검증**: 비교 분석에 사용할 문헌 수치의 정확성 확인

**팩트체크 호출 프로세스:**
```
해석/비교 중 검증 필요 항목 감지
    ↓
research-assistant 팩트체크 모드 호출
    ├─ web_search: "{keyword} site:pubmed.ncbi.nlm.nih.gov"
    ├─ 최소 2개 독립 출처 교차 검증
    └─ 판정: ✅ 확인 / ⚠️ 부분확인 / ❌ 불일치
    ↓
판정 결과를 현재 모드에 피드백
    ├─ ✅ → 그대로 진행
    ├─ ⚠️ → 조건 차이 명시 후 진행
    └─ ❌ → 해석/비교 내용 수정 후 진행
```

### Persistent Storage (체인 모드 전용)

**키 구조:**
- `disc:chain:{date}:{id}` — 체인 모드 전체 결과

## 디스커션 진행 가이드라인

### 질문 프레임워크
결과 해석 시 다음 질문들을 순서대로 고려:

1. **What**: 무엇이 관찰되었는가?
2. **Why**: 왜 이런 결과가 나왔는가?
3. **So what**: 이것이 왜 중요한가?
4. **What if**: 다른 조건이었다면 어떠했을까?
5. **What next**: 다음에 무엇을 해야 하는가?

### 비판적 사고 체크리스트
- 상관관계를 인과관계로 착각하지 않았는가?
- 대조군과의 비교가 적절한가?
- 재현성이 확보되었는가?
- 대안적 설명을 충분히 고려했는가?
- 데이터가 결론을 충분히 뒷받침하는가?
